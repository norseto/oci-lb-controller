# Plan Directory Rules

This directory stores execution plans by status.

## Directories

- `active/`: Plans that are currently in progress.
- `completed/`: Plans that have finished successfully.
- `failed/`: Plans that ended unsuccessfully.

## Usage

- Create each plan as a Markdown file in the directory that matches its current status.
- Move the plan file when its status changes.
- Keep plan filenames clear and descriptive.
- Keep completed and failed plans for traceability.
